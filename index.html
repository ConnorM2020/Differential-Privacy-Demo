About This Dashboard

This Differential Privacy Dashboard demonstrates how sensitive personal data can be protected through the application of noise, while still maintaining utility for statistical insights.

Patient Data Table

The central table displays anonymised patient data, including attributes such as age, town, health status, and vital signs. The data has been perturbed using Differential Privacy (DP), making it difficult to trace back any record to an individual while preserving statistical value.

Accuracy vs Privacy Chart (MAE)

The chart below visualizes the trade-off between privacy and data accuracy. It plots the Mean Absolute Error (MAE) as a function of the ε (epsilon) values. A lower epsilon means higher privacy but greater data distortion; higher epsilon values offer more accuracy but reduced privacy.

Location Privacy Map

The map displays randomised patient locations based on geographic noise. These positions are altered using DP to ensure location privacy. Each blue marker represents a patient, and the position becomes more randomised (less accurate) as the location ε-value is reduced.

Attribute & Location Sliders

Two interactive sliders control the ε-values for attribute and location noise:

  • ε (Attributes): Controls the amount of noise added to patient features like age or diagnosis.
  • ε (Location): Adjusts the randomness applied to the latitude and longitude values for geolocation privacy.

Use these sliders to observe real-time changes to both the table data and map display, and see how privacy-preserving mechanisms impact utility.

What is Epsilon (ε)?

Epsilon controls how much random noise is added to your query. Lower values mean stronger privacy but more noisy results.

  • ε = 0.1: Very private, very noisy
  • ε = 1.0: Balanced privacy and accuracy
  • ε = 3.0: Less private, more accurate

Try comparing the same query with ε = 0.1 and ε = 1.0 to see the effect.

Valid Inputs:

  • Towns: Belfast, Dublin, Cork
  • Diagnoses: Diabetes, Hypertension, Healthy
  • Epsilon (ε): Between 0.01 and 1.0 (smaller = more private, more noise)
